## 一、网站就是一家餐厅

你的教学辅助平台，相当于开了一家叫**"学霸餐厅"**的店：

- **学生**来吃饭（学习）
- **厨师**做菜（系统功能）
- **收银台**结账（登录验证）

### 餐厅有两个区域：大厅 + 后厨

| 名词 | 大白话 | 餐厅类比 |
|------|--------|---------|
| **前端** | 学生眼睛看到、手能点的页面 | 餐厅的装修、菜单、桌椅、服务员 |
| **后端** | 藏在服务器里的"大脑"，处理逻辑 | 后厨的厨师、冰箱、账本 |
| **数据库** | 存所有数据的地方 | 仓库，放着食材清单、会员信息、账本 |

## 二、大厅装修（前端）—— React + Ant Design

我们的大厅用了 **React** 技术。

**React 是什么？**
就像一个"积木工厂"。你不需要自己一块块木头锯，工厂给你现成的积木块（按钮、输入框、表格），你拼起来就行。

比如登录页面：
- 用户名输入框 = 一块积木
- 密码输入框 = 一块积木
- 登录按钮 = 一块积木
- React 就是把这些积木拼成一个页面

**Ant Design 是什么？**
一家专门做"高级积木"的厂商。我们用的侧边栏、表单、卡片，都是它家的现成积木，直接拿来用，不用自己画。

**TypeScript 是什么？**
就像积木说明书上标注了"这块积木只能放圆形孔"。如果你硬要放方形孔，它会提前报错。这样不容易拼错。

## 三、后厨运作（后端）—— FastAPI + Python

客人（学生）在大厅点单（点击按钮），订单怎么传到后厨？靠 **API**。

**API 是什么？**
就是大厅和后厨之间的"传菜口"。

比如学生点击"注册"：
1. 大厅把信息（用户名、密码）打包，塞进取餐口（发一个 API 请求）
2. 后厨收到，检查"这个用户名有没有人用过了？"
3. 检查完，回传结果"注册成功"或"用户名已存在"

**FastAPI 是什么？**
就是后厨的"自动化流水线"。你告诉它：
> "只要是塞到 `/register` 这个窗口的订单，走注册流程"

它自动帮你验证数据格式、生成文档、处理错误。

就是仓库管理员。你不需要自己手写复杂的账本格式（SQL 语句），你跟管理员说：
> "给我查一个叫张三的学生"

它帮你去翻账本，把结果拿回来。

## 五、门禁系统（JWT + RBAC）—— 怎么知道你是谁

### 登录 = 办会员卡

学生输入账号密码 → 后厨核对账本 → 对上了，发一张**会员卡（JWT Token）**。

这张卡里有加密信息："我是张三，我是学生"。

### 之后每次操作 = 刷卡

学生点击"进入课程页面"，大厅自动把会员卡塞进取餐口。后厨验卡：
- 卡是真的？→ 放行
- 卡过期了？→ 踢回登录页
- 卡上写着"学生"，但想进"管理员后台"？→ 拒绝（403）

**RBAC 是什么？**
就是"按角色给权限"。
- 学生卡 → 只能去大厅、普通座位
- 教师卡 → 能进包间、看厨房监控
- 管理员卡 → 哪都能去

## 六、三大模块是怎么回事

你的餐厅有三种服务：

### 1. 课程内容模块 = 自助点餐区

墙上挂着菜单（知识点列表），点哪道菜，屏幕上显示做法（学习资料）。菜单之间还有连线：先学会"炒蛋"才能学"番茄炒蛋"（知识图谱的前置关系）。

### 2. 编程实践模块 = 后厨实习区

给学生一个灶台（代码编辑器），让他自己炒一道菜（写代码）。旁边有个"试吃员"（代码沙箱），学生做完，试吃员尝一口，告诉他是咸了（报错）还是正好（运行成功）。

### 3. 智能体模块 = 智能服务员

学生做菜卡壳了，喊一声"这步怎么做？"。智能服务员（AI）看他现在做到哪一步了（上下文感知），不是直接给完整菜谱，而是先提示：
1. "你先想想火候"（思路引导）
2. "加一勺盐"（关键提示）
3. 实在不行再给完整步骤（参考代码）

## 七、学习数据闭环 = 餐厅的顾客分析系统

系统偷偷记录：
- 张三点了几道菜（学了哪些知识点）
- 每道菜吃了多久（学习时长）

- 问了多少次服务员（求助频率）

然后生成报告：
- **给张三看**：你的"循环"这道菜最弱，多练练（薄弱知识点清单）
- **给李老师看**：全班 30 人，20 个人卡在"递归"这（班级学情分析）

## 八、整个数据怎么流转（一条完整的故事线）

以学生"注册 → 登录 → 看课程 → 写代码 → 问 AI"为例：

```
学生打开网站
    ↓
大厅（React）展示登录页面
    ↓
输入账号密码，点击登录
    ↓
大厅把数据塞进取餐口（axios 发 API 请求）
    ↓
后厨（FastAPI）接到订单，查账本（SQLAlchemy 查 users 表）
    ↓
密码对上了 → 发会员卡（JWT Token）
    ↓
大厅存好会员卡（Zustand 状态管理 + localStorage）
    ↓
学生点击"课程内容"
    ↓
大厅刷会员卡，请求课程数据
    ↓
后厨验卡 → 放行 → 返回课程列表
    ↓
学生看到知识点，点击学"链表"
    ↓
学生进入编程实践，写代码
    ↓
点击运行 → 代码送到隔离厨房（Docker 沙箱）执行
    ↓
结果返回：成功/报错
    ↓
学生不会，问 AI → AI 结合当前题目给提示
    ↓
系统记录所有行为 → 生成学习报告
```
